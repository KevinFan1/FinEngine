FinEngine 商家对账说明

货号提取、红单匹配、货款关联与 Excel 透视复核指南

本文档把系统里的商家对账逻辑拆成可以在 Excel 中复刻的步骤。核心链路是： 从抖音动账商品名称提取货号，用货号和业务年月命中红单采购，再用采购单的商家、直播日期、直播间匹配红单货款， 最后基于匹配后的动账明细做透视汇总。

主要数据源 抖音动账明细、红单采购、红单货款、银行流水
核心键 货号、业务年月、商家、直播日期、直播间
汇总粒度 组织 + 我方主体 + 商家收款主体
适用场景 财务复核、手工排错、导出报表解释

整体流程

1 提取货号 从动账商品名称提取商品编码，多个独立货号用逗号分隔，组合货号保留加号。
2 lookup 采购 用货号 + 组织 + 店铺 + 业务年月，在最新红单采购中找到直播间、直播日期、大商家。
3 匹配货款 用采购单的商家、直播日期、直播间，匹配货款单的结算主体和收款商家。
4 透视汇总 用匹配后的动账明细按我方主体、收款商家聚合 GMV、直播款、银行流水差异。
系统计算时会先按同组织、同店铺、同业务年月读取最新一版红单。汇总页只纳入能匹配到采购和货款的动账记录；未匹配记录留在明细页用于排错。

重点注意事项

以下条件不满足时，后续匹配和透视结果都会不准

  • 红单采购里的 直播编号 / 新直播编码必须有关联货号，并且要能和动账商品名称提取出的货号对上。
  • 红单采购和红单货款的 直播日期必须统一口径，建议统一成 x月x日 或 x月x-y日，例如 4月1日、11月11-20日。
  • 不要在同一批数据里混用 2026-04-01、4/1、4.1、11/11-11/20 等不同写法；这些写法看起来一样，但匹配时可能对不上。
  • 货款匹配不是直接用货号找货款，而是先用货号找到采购，再用采购的 商家 + 直播日期 + 直播间 找货款。
  • 同一组织、同一店铺、同一业务年月如果上传多版红单，复核时要以 最新确认版本 为准。

业务复核时建议先检查红单采购的货号和直播日期，再检查红单货款的商家、直播日期、直播间。前面任一环节不统一，后面的收款商家和汇总金额都会跟着错。

数据表准备

建议把 Excel 工作簿中的四类数据转换成表格，表名固定后公式更稳定。

Excel 表名 来源 关键字段 用途
T_DZ 抖音动账明细 组织、店铺、业务年月、商品名称、商品编码、实收GMV、商户主体名称 提取货号并作为最终透视基础表
T_PUR 红单采购表 组织、店铺、业务年月、直播编号、新直播编码、商家、直播日期、直播间 用货号找到大商家和直播场次
T_PAY 红单货款表 组织、业务年月、商家、直播日期、直播间、结算主体、收款主体、收款商家、结算状态、已付 用采购场次找到收款商家和结算信息
T_BANK 银行流水 组织、业务年月、账户名称、对方户名、直播日期、支出金额、收入金额、流水净额 校验银行实际付款流出

系统中的业务年月来自动账核算年月或红单导入年月，通常用 YYYY-MM 展示，用 YYYYMM 作为匹配键更方便。 

=TEXT([@[业务年月]],"yyyymm")
=YEAR([@[业务年月]])*100+MONTH([@[业务年月]])

1. 提取货号

系统会先把商品名称统一成大写，再按“字母前缀 + 至少 3 位数字”提取货号。业务复核时可以直接使用系统导出的 商品编码 列，也可以用 Excel 公式辅助检查。

规则 说明 示例
基础货号 1 到 16 位字母 + 3 到 16 位数字 G16591、CAN1400899、ZC00100027
组合货号 组合内部保留 +，匹配时会再拆开 CAN123+CAN456
F/M 尾缀 只保留 -F 或 -M 这类尾缀 V33535-F、V61416-M
尺寸粘连 允许 mm/cm 后面直接跟货号 约33*13mmCAN3110492 提取 CAN3110492
材质黑名单 过滤金银材质编码，避免误认为货号 S925、AU750、PT950 不作为货号

Excel 公式：辅助提取货号

如果 Excel 可使用 REGEXEXTRACT 提取函数，可以在 T_DZ 增加辅助列 提取货号：

=LET(
  txt, UPPER(ASC([@[商品名称]])),
  raw, IFERROR(REGEXEXTRACT(txt,"([A-Z]{1,16}\d{3,16}(?:-[FM])?(?:\s*\+\s*[A-Z]{1,16}\d{3,16}(?:-[FM])?)*)"),""),
  code, SUBSTITUTE(raw," ",""),
  base, TEXTBEFORE(code&"-","-"),
  blacklist, {"S925","S990","S999","AG925","AG990","AG999","AU750","AU916","AU999","G750","G999","PT900","PT950","PT990","PT999","K9","K10","K14","K18","K22","K24"},
  IF(ISNUMBER(XMATCH(base,blacklist)),"",code)
)
注意：上面的公式用于业务复核，不能替代系统最终匹配结果。大量复核时建议优先使用系统导出的 商品编码 列。

没有提取函数时

如果 Excel 不支持上述函数，建议直接使用系统导出的 商品编码 列；少量异常数据可以人工补货号。 人工补货号时，必须和红单采购里的 直播编号 / 新直播编码 保持一致。

拆分组合货号

系统匹配采购时会把逗号和加号都拆开。Excel 复核建议增加 货号数组 辅助逻辑：

=TEXTSPLIT(SUBSTITUTE(SUBSTITUTE([@[提取货号]],"＋",","),"+",","),",")

2. 用货号 + 年月 lookup 红单采购单信息

系统先按组织、店铺、业务年月读取最新一版红单，再在红单采购表中建立货号索引。采购索引来源优先使用 新直播编码， 同时也会看原始 直播编号。命中后得到大商家、直播间、直播日期等字段。

本步骤必须先确认

  • 红单采购的 直播编号 / 新直播编码 要填写货号，不能只写直播场次备注。
  • 动账提取出来的货号，要能在红单采购中找到；否则大商家、直播间、直播日期都会为空。
  • 同一个货号如果在同月同店铺出现多次，需要人工确认是否重复或是否属于组合货号。

采购表辅助列

辅助列 公式 作用
采购年月 =TEXT([@[业务年月]],"yyyymm") 让采购和动账按同一账期匹配
采购货号 =UPPER(TRIM(IF([@[新直播编码]]<>"",[@[新直播编码]],[@[直播编号]]))) 优先使用新直播编码；没有时使用直播编号
采购匹配键 =[@组织]&"|"&[@店铺]&"|"&[@采购年月]&"|"&[@采购货号] 组织、店铺、年月、货号组成唯一查找键

动账表 lookup 采购

在 T_DZ 增加 动账年月 和 采购匹配键：

=TEXT([@[业务年月]],"yyyymm")
=[@组织]&"|"&[@店铺]&"|"&[@动账年月]&"|"&[@[提取货号]]

拉取大商家、直播间、直播日期

=XLOOKUP([@[采购匹配键]], T_PUR[采购匹配键], T_PUR[商家], "")
=XLOOKUP([@[采购匹配键]], T_PUR[采购匹配键], T_PUR[直播间], "")
=XLOOKUP([@[采购匹配键]], T_PUR[采购匹配键], T_PUR[直播日期], "")
如果一个商品名称提取出多个货号，系统会按顺序尝试，命中第一个采购货号就停止。Excel 复核时可先展开为多行再 lookup，或者用 TEXTSPLIT + MAP 自定义处理。

多货号复核公式

下面公式会把一行动账中的多个货号逐个尝试，返回第一个匹配到的采购商家：

=LET(
  codes, TEXTSPLIT(SUBSTITUTE(SUBSTITUTE([@[提取货号]],"＋",","),"+",","),","),
  keys, [@组织]&"|"&[@店铺]&"|"&[@动账年月]&"|"&codes,
  hits, XLOOKUP(keys, T_PUR[采购匹配键], T_PUR[商家], ""),
  IFERROR(INDEX(FILTER(hits,hits<>""),1),"")
)

3. 用采购单的直播间、直播日期、货号匹配货款信息

系统的货款匹配不是直接用货号，而是先通过货号找到采购行，再用采购行上的 商家 + 直播日期 + 直播间 去匹配红单货款表。命中后得到结算主体、收款主体、收款商家、已付金额等信息。

标准化规则

字段 系统处理 Excel 建议
商家 去首尾空格，转大写 =UPPER(TRIM([@商家]))
直播日期 日期建议统一为 4月1日、11月11-20日 等红单展示口径 必须统一红单采购和货款的日期文本
直播间 去首尾空格，转大写 =UPPER(TRIM([@直播间]))

货款表辅助列

货款年月 = TEXT([@[业务年月]],"yyyymm")
货款匹配键 = [@组织]&"|"&[@货款年月]&"|"&UPPER(TRIM([@商家]))&"|"&[@直播日期]&"|"&UPPER(TRIM([@直播间]))

动账表匹配货款

动账表先用上一步得到的采购字段生成 货款匹配键：

=[@组织]&"|"&[@动账年月]&"|"&UPPER(TRIM([@[大商家]]))&"|"&[@[直播日期]]&"|"&UPPER(TRIM([@[直播间]]))

拉取货款字段

我方主体 = XLOOKUP([@[货款匹配键]], T_PAY[货款匹配键], T_PAY[结算主体], "")
商家收款主体 = XLOOKUP([@[货款匹配键]], T_PAY[货款匹配键], T_PAY[收款主体], "")
收款商家 = XLOOKUP([@[货款匹配键]], T_PAY[货款匹配键], T_PAY[收款商家], "")
已付 = XLOOKUP([@[货款匹配键]], T_PAY[货款匹配键], T_PAY[已付], 0)
结算状态 = XLOOKUP([@[货款匹配键]], T_PAY[货款匹配键], T_PAY[结算状态], "")
注意：汇总复核时要重点关注红单货款的 结算状态。涉及付款调整时，建议优先复核 已结算 的货款行。

4. 用匹配的动账信息进行透视

匹配完成后，每行动账都有货号、大商家、直播日期、直播间、收款商家等字段。系统会计算直播款、BIC 分摊和运费险分摊， 然后按汇总粒度聚合。

动账明细计算列

字段 系统口径 Excel 公式
直播款 实收GMV * 0.7，四舍五入到 2 位小数 =ROUND([@[实收GMV]]*0.7,2)
分摊 BIC 本行 GMV / 当月总 GMV * 当月 BIC =ROUND([@[实收GMV]]/SUMIFS(T_DZ[实收GMV],T_DZ[动账年月],[@动账年月])*当月BIC,2)
分摊运费险 本行 GMV / 当月总 GMV * 当月运费险 =ROUND([@[实收GMV]]/SUMIFS(T_DZ[实收GMV],T_DZ[动账年月],[@动账年月])*当月运费险,2)
匹配状态 货号、采购、货款都命中为已匹配，否则未匹配 =IF(AND([@[提取货号]]<>"",[@[大商家]]<>"",[@[收款商家]]<>""),"已匹配","未匹配")

透视表配置

区域 字段 说明
筛选器 组织、店铺、业务年月、匹配状态 建议只看 已匹配，未匹配另做异常表
行 我方主体、商家收款主体 系统汇总口径接近 组织 + 我方主体 + 收款商家
值 实收GMV、直播款、分摊BIC、分摊运费险、动账行数 全部使用求和，动账行数使用计数
列 业务年月或银行流水状态 跨月复核时使用

不用透视表的 SUMIFS 汇总公式

如果建立了汇总表 T_SUM，每行包含 组织、业务年月、我方主体、商家收款主体，可用公式：

实收GMV =
SUMIFS(T_DZ[实收GMV], T_DZ[组织], [@组织], T_DZ[动账年月], [@业务年月], T_DZ[我方主体], [@我方主体], T_DZ[收款商家], [@商家收款主体], T_DZ[匹配状态], "已匹配")

应付商家净额 =
SUMIFS(T_DZ[直播款], T_DZ[组织], [@组织], T_DZ[动账年月], [@业务年月], T_DZ[我方主体], [@我方主体], T_DZ[收款商家], [@商家收款主体], T_DZ[匹配状态], "已匹配")

汇总金额字段

应付货款余额 = 应付商家净额 + 期初余额 - 冲减业务费用 - 其他冲减金额
未付流水 = 应付货款余额 - 已付流水
银行付款差 = 未付流水 - 银行流水汇总
银行流水状态 = IF(银行流水汇总=0,"待匹配",IF(银行付款差=0,"已匹配","有差异"))

银行流水校验

银行流水不是货号匹配链路的第一步，但会进入最终汇总校验。系统只纳入用途、备注或附言里包含“直播”的流水行，并且只统计 流水净额 > 0 的付款流出。

银行流水辅助列

流水净额 = ROUND([@[支出金额]]-[@[收入金额]],2)
银行匹配键 = [@组织]&"|"&UPPER(TRIM([@[账户名称]]))&"|"&UPPER(TRIM([@[对方户名]]))&"|"&[@直播日期]

红单货款银行匹配键

=[@组织]&"|"&UPPER(TRIM([@[结算主体]]))&"|"&UPPER(TRIM([@[收款商家]]))&"|"&[@直播日期]

能桥接到红单货款时，银行流水会再按该货款行关联到的汇总组 直播款 权重分摊，避免同一笔付款被多个货号重复计入。 Excel 手工复核如果不处理分摊，可以先按主体和收款商家粗略校验，再对差异行逐笔追踪。

异常检查

检查项 Excel 公式或方法 处理建议
未提取到货号 =FILTER(T_DZ,T_DZ[提取货号]="") 检查商品名称是否缺编码，或编码被写成非系统支持格式
货号未匹配采购 =FILTER(T_DZ,(T_DZ[提取货号]<>"")*(T_DZ[大商家]="")) 检查红单采购是否同年月、同店铺、同组织，直播编号是否填写货号
采购未匹配货款 =FILTER(T_DZ,(T_DZ[大商家]<>"")*(T_DZ[收款商家]="")) 检查商家、直播日期、直播间三项是否和红单货款表一致
同一采购货号重复 =COUNTIF(T_PUR[采购匹配键],[@采购匹配键]) 系统会取最新红单内首次出现的采购行，重复时建议人工确认
银行流水待匹配 筛选 银行流水状态 = 待匹配 检查账户名称、对方户名、直播日期是否能桥接红单货款
最低交付校验：未提取货号、未匹配采购、未匹配货款三张异常表都要能解释；汇总透视的实收 GMV 和系统汇总页同条件金额应一致。

系统明细导出核心字段

字段 含义 来源
商品编码 动账商品名称提取出来的货号 系统提取结果
大商家名称 通过货号命中的红单采购商家 红单采购
收款商家 通过采购场次命中的红单货款收款商家 红单货款

系统汇总导出核心字段

字段 计算口径
实收GMV 匹配明细的 GMV 求和
应付商家净额 匹配明细的直播款求和
应付货款余额 应付商家净额 + 期初余额 - 冲减业务费用 - 其他冲减金额

容易混淆的口径

  • 大商家名称 来自红单采购的 商家，用于说明货号归属。
  • 收款商家 来自红单货款，用于最终收款主体和银行流水校验。
  • 系统汇总时的 我方主体 优先使用动账明细里的 商户主体名称，没有时才退回结算主体或店铺名。
  • 采购匹配使用货号；货款匹配使用采购行上的 商家 + 直播日期 + 直播间。
  • 银行流水只纳入付款流出，不把收入或冲正类流水计入银行付款汇总。
